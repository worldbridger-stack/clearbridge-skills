CLEARBRIDGE DESIGN SYSTEM
Развёрнутая спецификация для развития

ВВЕДЕНИЕ
Данный документ фиксирует развёрнутую дизайн-систему ClearBridge как методологический и визуальный стандарт.
Он предназначен для работы, развития, передачи дизайнерам, верстальщикам и использования как канон при масштабировании трилогии, курсов и платформы.

I. ИДЕОЛОГИЧЕСКИЙ УРОВЕНЬ
1. Назначение дизайн-системы
Дизайн-система ClearBridge существует не для украшения контента, а для обеспечения работы с мышлением читателя.
Книга рассматривается как инструмент, а не как носитель текста. Каждая страница должна помогать предпринимателю думать, фиксировать, анализировать и принимать решения.
Визуал не дополняет текст, а является равноправным носителем смысла.
2. Принципы системы
Архитектурность означает, что структура важнее формы: если элемент красив, но не выполняет функцию — он недопустим.
Повторяемость позволяет читателю быстро ориентироваться и снижает когнитивную нагрузку.
Масштабируемость обеспечивает перенос методологии в другие форматы.
Минимум украшательства исключает эмоциональный шум.
Ясность всегда приоритетнее эстетики.
Каждый визуальный элемент выполняет строго одну функцию.
3. Роль дизайна в ClearBridge
Дизайн помогает предпринимателю выйти из хаоса мышления.
Он формирует единый язык между владельцем, командой и консультантом.
Дизайн подсказывает, как мыслить структурно, а не как читать книгу.

II. СЕМАНТИЧЕСКИЙ УРОВЕНЬ
4. Семантика цвета
Цвет используется как носитель смысла.
Каждый цвет соответствует типу мышления: аналитическому, ценностному, операционному, системному.
Цвета не смешиваются произвольно и не используются для декора.
Один цвет — один смысл во всей трилогии.
5. Семантика форм
Форма определяет характер информации.
Прямоугольники используются для структур и моделей.
Линии показывают связи и потоки.
Стрелки обозначают направление изменений и трансформаций.
Рамки фиксируют границы системы.
Пунктир используется для гипотез и временных конструкций.
6. Семантика иконок
Иконки представляют тип мыслительной операции.
Они помогают мгновенно распознавать тип блока.
Иконки не несут декоративной функции и всегда повторяемы.

III. СТРУКТУРНЫЙ УРОВЕНЬ СТРАНИЦЫ
7. Архитектура страницы
Каждая страница имеет чёткую роль.
На странице должна быть одна главная мысль.
Второстепенные элементы не конкурируют с основной.
Визуальный якорь помогает быстро считать смысл.
Предусматривается пространство для заметок и работы.
8. Типы страниц
Концептуальные страницы вводят понятия.
Объясняющие раскрывают логику.
Практические направляют к действию.
Диагностические помогают оценить состояние бизнеса.
Переходные связывают главы и книги.
Рефлексивные позволяют зафиксировать выводы.

IV. ТИПОЛОГИЯ КОНТЕНТНЫХ БЛОКОВ
9. Базовые блоки
Concept — фиксирует определение.
Insight — объясняет значимость.
Framework — визуализирует модель.
Practice — описывает действие.
Question — задаёт вопросы бизнесу.
Checklist — проверяет полноту.
Example — даёт конкретику.
Anti-pattern — показывает ошибку.
Warning — предупреждает о риске.
Transition — связывает логику.
10. Поведение блоков
Каждый блок имеет фиксированную структуру.
Блоки визуально узнаваемы.
Поведение блоков одинаково во всей системе.
Блоки могут использоваться независимо от контекста.

V. ТИПОГРАФИКА
11. Иерархия текста
Иерархия позволяет считывать структуру без чтения.
Заголовки управляют вниманием.
Основной текст лаконичен.
Микротекст используется для пояснений.
12. Роль шрифта
Шрифт подбирается по функциональности.
Один шрифт для чтения.
Один для структур и схем.
Красота не является критерием выбора.

VI. ВИЗУАЛЬНЫЕ АРТЕФАКТЫ
13. Диаграммы и схемы
Все диаграммы являются рабочими инструментами.
Используются BMC, VPC, CJM, архитектурные схемы.
Каждая схема объясняет поток ценности.
14. Правила построения схем
Единый стиль во всей системе.
Единые обозначения.
Подписи обязательны.
Минимум визуального шума.

VII. НАВИГАЦИЯ И СВЯЗИ
15. Навигация внутри книги
Читатель всегда понимает, где он находится.
Показывается цель главы.
Обозначается логика перехода.
16. Связь книг трилогии
Книги связаны визуально и логически.
Читатель понимает траекторию роста.

VIII. МЕТА-УРОВЕНЬ
17. Уровни глубины
Контент может читаться с разных уровней.
Предприниматель — для понимания.
Сотрудник — для выполнения.
Консультант — для применения.
Архитектор — для развития системы.
18. Режимы использования
Книга может использоваться для чтения.
Для диагностики.
Для консалтинга.
Для обучения.

IX. КРОСС-МЕДИЙНОСТЬ
19. Переиспользование
Контент должен легко переноситься в презентации, курсы, платформы.
20. Digital-ready
Контент проектируется с учётом цифровых форматов.

X. УПРАВЛЕНИЕ СИСТЕМОЙ
21. Версионирование
Система развивается итеративно.
22. Правила изменений
Изменения фиксируются централизованно.

XI. АНТИ-ЦЕЛИ
23. Запрещённые практики
Любой декоративный дизайн без смысла.
Разрозненные стили.
Иллюстрации ради визуала.

XII. ЭТАЛОНЫ
24. Эталонные артефакты
Фиксируются эталонные страницы и блоки.

XIII. РЕЗУЛЬТАТ
ClearBridge — это операционная методология.
Дизайн является её неотъемлемой частью.
Система готова к развитию в платформу.
